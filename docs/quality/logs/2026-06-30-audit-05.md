### 🔴 P0-1：Server 端测试覆盖率 3.6%（28 模块仅 1 有测试）

`server/src/` 共 28 个源模块，仅 `store/nodePoolRepository.ts` 有 16 个单测。以下核心模块**零测试**：

## 4. 🛠️ 长期维护与优化行动指南

**第一梯队（立即做，低风险）**
1. **修复 Step3Clean 3 个预存 TS 错误 + 2 个 lint 错误**：删未用 import（`parseSSE`/`ProviderNode`），修正 `streamSingleChannel` 参数。已在 HANDOFF 多轮标注为「预存」但未修——应在新 audit 前清零。
2. **消除 persistence.ts 的 20 处 `.catch(() => {})`**：改为 `console.error` + toast 通知（`message.error`），至少保留错误日志可追溯。
3. **importSession.ts 空 catch 改日志**：3 处 `catch {}` unlinkSync 至少加 `console.warn`。

**第二梯队（1–2 迭代，中风险）**
1. **Server 核心模块加测试**：优先 `llmClient.ts`（mock Provider）→ `db.ts`（`:memory:` 建表/CRUD）→ `creation.*.ts`（mock SSE 流）
2. **拆分 3 个巨组件**：Step3Clean → 4 文件，ImmersiveReader → 4 文件，m2-cards → 3 文件。每组件拆分独立 commit，互不阻塞。
3. **gifUtils CDN Worker 本地化**：打包时将 `gif.worker.js` 复制到 `public/`，离线可用。
4. **补 16 个目录 barrel export**：批量生成 `index.ts` re-export，零破坏性。

**第三梯队（需先出设计稿）**
1. **页面级集成测试**：Playwright/Cypress E2E 覆盖 M0→M5 全创作链路。
2. **Electron 42 迁移**：评估 breaking changes（Window API / process model），在 `ELECTRON.md` 补充迁移指南。
3. **TypeScript 6 升级**：评估 `--erasableSyntaxOnly` 等新特性影响，优先在 server 侧验证。

**贯穿纪律**
- 单文件 > **800 行**触发拆分评审（当前 3 文件超标）
- `.catch(() => {})` 视为 **lint error** 级别：新增禁止，存量逐批消
- 后端模块新增/修改必须带对应 `*.test.ts`
- 依赖升级：每月跑 `npm outdated`，minor/patch 立即升，major 需走设计稿
